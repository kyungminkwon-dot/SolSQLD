# Project: SolSQLD (SQLD 실습 및 모의고사 플랫폼)

## 1. 프로젝트 개요
- [cite_start]**서비스 명**: SolSQLD [cite: 65, 71]
- [cite_start]**핵심 목표**: Oracle 환경 기반의 실시간 SQL 실습 및 실제 시험장 환경을 재현한 모의고사 환경 제공 [cite: 68, 69, 296]
- [cite_start]**기획 및 대표**: 권경민 [cite: 73]

## 2. 주요 페이지 구성
- [cite_start]**메인 화면**: 서비스 소개 및 핵심 메뉴(모의고사, SQL 실습) 진입로 [cite: 65-79]
- [cite_start]**회원가입/로그인**: 팝업 UI로 구현 [cite: 80-90, 294]
- [cite_start]**개인 대시보드**: 최근 푼 문제 및 과목별 정답률 시각화 [cite: 91-112, 294]
- **모의고사 시스템**:
    - [cite_start]리스트: 회차별 문항 수(50문항), 평균 난이도, 풀이 여부 표시 [cite: 116-127, 295]
    - [cite_start]풀이 화면: **A4 스케일 레이아웃**, 문제 번호, 남은 시간 카운트다운, 사이드 메모장 [cite: 133-154, 295]
    - [cite_start]결과 및 해설: 합격/불합격 판정, 오답 및 해설 확인 [cite: 165-168, 295]
- **SQL 실습 환경**:
    - [cite_start]검색 및 필터: 문제명 검색, 난이도/정답률 정렬 [cite: 202-211, 296]
    - [cite_start]쿼리 편집기: SQL 구문 강조(Highlighting) 지원 [cite: 239, 296]
    - [cite_start]결과 출력: 실시간 테이블 결과 또는 에러 메시지 출력 [cite: 240, 264, 296]

## 3. 핵심 프론트엔드 로직
- [cite_start]**닉네임 자동 설정**: 회원가입 시 닉네임 미입력 시 이메일의 @ 앞부분을 자동 할당 [cite: 294]
- [cite_start]**약관 동의 절차**: '전문 보기' 클릭 후 스크롤을 하단까지 내려야 동의 버튼 활성화 [cite: 294]
- [cite_start]**UX 요소**: 사이드 메모장 입력 내용 실시간 유지 및 SQL 결과 팝업 피드백 [cite: 270, 277, 296]

## 4. 데이터 연동 및 이벤트 로깅
- [cite_start]**ERD 준수**: User, Problem, Submission, Exam_History 테이블 구조 기반 상태 관리 [cite: 297-387]
- [cite_start]**이벤트 로그 (11개 필수 트리거)**: [cite: 388, 389]
    1. 회원가입/로그인 성공 및 최초 접속 로그
    2. SQL 실행/제출 시 쿼리문 및 정답 여부 기록
    3. 객관식 보기 선택 및 모의고사 시작/최종 제출 기록
    4. 사이드 메모장 입력 텍스트 유지 관리
    5. 정답 처리 시 유저 포인트 업데이트 (+10점)
    6. 문제 풀이 종료 시 누적 통계(정답률 등) 업데이트
