# ESLint + Prettier + GitHub Actions CI 구성 계획

**작성일:** 2026-03-01
**대상 프로젝트:** SolSQLD (React + TypeScript, Vite)

---

## Context

SolSQLD 프로젝트에 코드 품질 도구가 전혀 없는 상태. ESLint(린트) + Prettier(포매팅) + GitHub Actions(PR 자동 검사)를 추가하여 develop/main 브랜치 전략과 연동되는 CI 파이프라인을 구축한다.

---

## 생성/수정할 파일

| 파일 | 동작 | 목적 |
|------|------|------|
| `SolSQLD/eslint.config.js` | 생성 | ESLint 9 flat config (TS + React Hooks + Prettier 충돌 방지) |
| `SolSQLD/.prettierrc` | 생성 | Prettier 규칙 (싱글쿼트, 세미콜론, 2스페이스 등 기존 스타일 유지) |
| `SolSQLD/.prettierignore` | 생성 | dist/node_modules 제외 |
| `SolSQLD/package.json` | 수정 | devDependencies 7개 추가 + lint/format 스크립트 추가 |
| `.github/workflows/ci.yml` | 생성 | PR시 format check → lint → type check 자동 실행 |

---

## 설치할 패키지 (devDependencies)

```
eslint, @eslint/js, typescript-eslint, eslint-plugin-react-hooks,
eslint-plugin-react-refresh, eslint-config-prettier, prettier
```

---

## 주요 설정

### Prettier — 기존 코드 스타일과 일치

- `singleQuote: true`
- `semi: true`
- `tabWidth: 2`
- `trailingComma: "es5"`
- `printWidth: 100`

### ESLint — flat config (ESM)

- `@eslint/js` recommended + `typescript-eslint` recommended
- `react-hooks` + `react-refresh` 플러그인
- `_prefix` 변수 unused 허용 (`argsIgnorePattern: '^_'`)
- `eslint-config-prettier`를 마지막에 적용하여 포매팅 규칙 충돌 방지

### GitHub Actions CI (`ci.yml`)

- **트리거:** `pull_request` + `push` to `main`, `develop`
- `working-directory: SolSQLD` 설정
- **순서:** checkout → Node 20 setup (npm cache) → `npm ci` → `prettier --check` → `eslint` → `tsc --noEmit`

### package.json 스크립트 추가

| 스크립트 | 명령어 | 용도 |
|----------|--------|------|
| `lint` | `eslint .` | 린트 검사 |
| `lint:fix` | `eslint . --fix` | 린트 자동 수정 |
| `format` | `prettier --write .` | 포매팅 적용 |
| `format:check` | `prettier --check .` | 포매팅 검사 (CI용) |
| `check` | `npm run format:check && npm run lint && tsc --noEmit` | CI와 동일한 로컬 검증 |

---

## 커밋 전략 (3단계)

1. **도구 설정 커밋** — config 파일 생성 + 패키지 설치
2. **포매팅 커밋** — `prettier --write .` (순수 스타일 변경, 로직 무변경)
3. **린트 수정 커밋** — ESLint 위반 사항 수정

---

## 검증

1. `npm run check` 로컬 실행하여 format + lint + typecheck 통과 확인
2. `npm run build` 정상 빌드 확인
3. develop 브랜치에서 작업 후 PR 생성하여 GitHub Actions 동작 확인
